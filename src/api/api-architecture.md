# ml-sharp API 功能结构文档

## 1. 项目概述

ml-sharp API 是一个基于 FastAPI 构建的 RESTful API 服务，用于从 2D 图像生成 3D 高斯点云（3D Gaussians）。该服务提供了模型推理、结果渲染和 Web UI 等核心功能，支持多种设备运行，并具有模型缓存机制。

## 3. 核心模块功能

### 3.1 主应用模块 (`src/api/app.py`)

| 功能 | 描述 | 实现细节 |
|------|------|----------|
| 应用初始化 | 创建 FastAPI 应用实例，配置基本信息 | 设置标题、描述、版本、文档路径 |
| CORS 配置 | 允许跨域请求，支持所有来源和方法 | 配置 `CORSMiddleware` |
| 静态文件服务 | 提供生成结果的访问服务 | 挂载 `/results` 路径，指向 `/tmp/ml-sharp/results` 目录 |
| 路由管理 | 注册 API 端点和 Web UI | 包含健康检查、根端点和预测路由 |
| Web UI 集成 | 挂载 Gradio 应用 | 提供可视化操作界面 |

### 3.2 预测端点 (`src/api/endpoints/predict.py`)

| 功能 | 描述 | 实现细节 |
|------|------|----------|
| 图像验证 | 检查上传文件是否为有效图像 | 验证文件类型和扩展名 |
| 图像预处理 | 处理图像 EXIF 数据、旋转和焦距计算 | 提取 EXIF 信息，自动旋转图像，计算像素焦距 |
| 设备管理 | 根据请求选择运行设备 | 支持 CPU、MPS、CUDA 设备 |
| 模型推理 | 从图像生成 3D 高斯点云 | 使用 `RGBGaussianPredictor` 模型 |
| 结果保存 | 保存生成的 3D 高斯点云为 PLY 文件 | 生成唯一文件名，保存到结果目录 |
| 视频渲染 | 可选的 3D 高斯点云视频渲染 | 仅在 CUDA 设备上支持，生成旋转视角视频 |

### 3.3 渲染模块 (`src/api/endpoints/render.py`)

| 功能 | 描述 | 实现细节 |
|------|------|----------|
| 相机轨迹生成 | 创建用于渲染的相机运动轨迹 | 生成围绕场景的圆周轨迹 |
| 帧渲染 | 渲染 3D 高斯点云为 2D 帧 | 使用 `gsplat` 库进行高效渲染 |
| 视频保存 | 将渲染帧合成为视频文件 | 支持自定义帧数和帧率 |

### 3.4 数据模型 (`src/api/schemas/predict.py`)

| 模型 | 用途 | 主要字段 |
|------|------|----------|
| `PredictRequest` | 预测请求参数 | `device` (设备选择)、`render` (是否渲染)、`checkpoint_path` (自定义模型路径) |
| `PredictResult` | 预测结果数据 | `gaussians_ply_path` (PLY文件路径)、`render_video_path` (视频路径)、`inference_time` (推理时间)、`image_dimensions` (图像尺寸)、`num_gaussians` (高斯点数量) |
| `PredictResponse` | 预测响应包装 | `success` (成功状态)、`message` (响应消息)、`result` (结果数据)、`error` (错误信息) |
| `HealthResponse` | 健康检查响应 | `status` (健康状态)、`models_loaded` (已加载模型数)、`timestamp` (时间戳) |

### 3.5 依赖管理 (`src/api/dependencies.py`)

| 功能 | 描述 | 实现细节 |
|------|------|----------|
| 模型缓存 | 管理已加载的模型实例 | 实现 `ModelCache` 类，提供模型获取和缓存功能 |
| 依赖注入 | 为 API 端点提供依赖项 | 使用 FastAPI 的依赖注入系统 |

## 4. API 端点详情

### 4.1 核心预测端点

| 端点 | 方法 | 描述 | 请求体 | 响应 |
|------|------|------|--------|------|
| `/api/predict` | `POST` | 从图像生成 3D 高斯点云 | 图像文件 + `PredictRequest` 参数 | `PredictResponse` |

**请求示例：**
```http
POST /api/predict
Content-Type: multipart/form-data

file: <image.jpg>
device: cuda
render: true
```

**响应示例：**
```json
{
  "success": true,
  "message": "Prediction completed successfully.",
  "result": {
    "gaussians_ply_path": "/results/abc123_image_ply",
    "render_video_path": "/results/abc123_image_mp4",
    "inference_time": 1.234,
    "image_dimensions": {"height": 1080, "width": 1920},
    "num_gaussians": 100000
  },
  "error": null
}
```

### 4.2 辅助端点

| 端点 | 方法 | 描述 | 响应 |
|------|------|------|------|
| `/health` | `GET` | 检查 API 健康状态 | 包含状态、已加载模型数和时间戳 |
| `/` | `GET` | API 根端点 | 提供基本信息和文档链接 |
| `/webui` | `GET` | Gradio Web UI | 可视化操作界面 |
| `/docs` | `GET` | Swagger 文档 | 交互式 API 文档 |
| `/redoc` | `GET` | ReDoc 文档 | 静态 API 文档 |

## 5. 技术特点与优势

### 5.1 高性能设计

- **模型缓存机制**：避免重复加载模型，提高推理效率
- **多设备支持**：自动检测并利用可用的硬件加速（CUDA/MPS）
- **异步处理**：使用 FastAPI 的异步特性，提高并发处理能力
- **高效渲染**：使用 `gsplat` 库进行实时 3D 高斯渲染

### 5.2 易用性设计

- **RESTful API**：遵循 RESTful 设计原则，易于集成
- **自动生成文档**：提供 Swagger 和 ReDoc 文档，方便开发者使用
- **Web UI 集成**：提供可视化操作界面，适合非技术用户
- **清晰的响应格式**：统一的响应结构，包含状态、消息和数据

### 5.3 可扩展性设计

- **模块化架构**：清晰的模块划分，便于功能扩展
- **自定义模型支持**：允许使用自定义模型检查点
- **灵活的配置选项**：支持设备选择、渲染开关等配置

## 6. 工作流程

1. **请求接收**：API 接收图像文件和预测参数
2. **图像预处理**：
   - 验证图像文件
   - 提取 EXIF 信息
   - 自动旋转图像
   - 计算像素焦距
3. **模型推理**：
   - 从缓存获取或加载模型
   - 运行模型推理，生成 3D 高斯点云
4. **结果处理**：
   - 保存 3D 高斯点云为 PLY 文件
   - （可选）渲染 3D 高斯点云为视频
5. **响应返回**：返回包含结果路径和元数据的响应

## 7. 部署与配置

### 7.1 环境要求

- Python 3.8+
- FastAPI
- PyTorch
- Gradio
- 其他依赖项（见项目 `requirements.txt`）

### 7.2 启动服务

```bash
python start_server.py
```

### 7.3 主要配置项

| 配置项 | 描述 | 默认值 |
|--------|------|--------|
| 结果目录 | 生成结果的保存目录 | `/tmp/ml-sharp/results` |
| API 前缀 | API 端点的 URL 前缀 | `/api` |
| 文档路径 | Swagger 和 ReDoc 文档路径 | `/docs` 和 `/redoc` |
| Web UI 路径 | Gradio UI 路径 | `/webui` |
| CORS 配置 | 允许的来源、方法和头部 | 允许所有（生产环境建议限制） |

## 9. 总结

ml-sharp API 是一个功能完整、设计良好的 3D 高斯点云生成服务，具有高性能、易用性和可扩展性等特点。它提供了 RESTful API 和 Web UI 两种交互方式，支持多种设备运行，并具有模型缓存、结果保存和视频渲染等功能。该 API 适合集成到各种应用中，用于从 2D 图像生成高质量的 3D 高斯点云。

通过清晰的模块化设计和详细的文档，开发者可以轻松理解和使用该 API，同时也便于后续的功能扩展和维护。
